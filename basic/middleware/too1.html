Utility Middleware Explained

Middleware functions are the building blocks of web applications that handle tasks like parsing requests, authentication, logging, and more. They sit between the request and response in the application cycle.

Logging Middleware

Records information about incoming requests for debugging and monitoring purposes.

const loggingMiddleware = (req, res, next) => {
  console.log(`${new Date().toISOString()} - ${req.method} ${req.path}`);
  next();
};

Use Case

Track all incoming requests to your server for debugging and analytics.

Authentication Middleware

Verifies user credentials before allowing access to protected routes.

const authMiddleware = (req, res, next) => {
  const token = req.header('Authorization');
  if (!token) return res.status(401).send('Access denied');
  try {
    const verified = jwt.verify(token, process.env.JWT_SECRET);
    req.user = verified;
    next();
  } catch (err) {
    res.status(400).send('Invalid token');
  }
};

Use Case

Protect routes that should only be accessible to logged-in users.

Error Handling Middleware

Centralizes error handling logic for cleaner code and consistent error responses.

const errorHandler = (err, req, res, next) => {
  console.error(err.stack);
  res.status(500).json({
    error: 'Something went wrong!',
    message: process.env.NODE_ENV === 'production'
      ? 'Internal server error' : err.message
  });
};

Use Case

Provide consistent error responses and prevent server crashes from unhandled errors.

Body Parsing Middleware

Extracts data from incoming requests and makes it easily accessible.

const bodyParser = (req, res, next) => {
  let data = '';
  req.on('data', chunk => data += chunk);
  req.on('end', () => {
    try {
      req.body = JSON.parse(data);
      next();
    } catch (err) {
      res.status(400).send('Invalid JSON');
    }
  });
};

Use Case

Parse JSON payloads from client applications to easily access request data.

CORS Middleware

Enables Cross-Origin Resource Sharing for web applications making requests from different domains.

const corsMiddleware = (req, res, next) => {
  res.header('Access-Control-Allow-Origin', '*');
  res.header('Access-Control-Allow-Headers',
    'Origin, X-Requested-With, Content-Type, Accept');
  res.header('Access-Control-Allow-Methods',
    'GET, POST, PUT, DELETE, OPTIONS');
  next();
};

Use Case

Allow frontend applications from different domains to access your API endpoints.

Rate Limiting Middleware

Prevents abuse by limiting how many requests a client can make in a given time period.

const rateLimit = {};
const rateLimiter = (req, res, next) => {
  const ip = req.ip;
  const now = Date.now();
  const windowStart = now - 60000; // 1 minute window

  if (!rateLimit[ip]) rateLimit[ip] = [];
  rateLimit[ip] = rateLimit[ip].filter(time => time > windowStart);

  if (rateLimit[ip].length >= 100) {
    return res.status(429).send('Too many requests');
  }
  rateLimit[ip].push(now);
  next();
};

Use Case

Protect your API from abuse, DoS attacks, and excessive resource consumption.
